СНАЧАЛА НАСТРОЙКА ГИТ - УКАЗЫВАЕМ ИМЯ АВТОРА И ЭЛЕКТРОННУЮ ПОЧТУ

СОЗДАЕМ НОВУЮ ПАПКУ И ПЕРЕХОДИМ В НЕЕ ЧЕРЕЗ ТЕРМИНАЛ
mkdir project
cd project

СОЗДАЕМ ФАЙЛ В ПАПКЕ
touch index.html

СОЗДАДИМ ЕЩЕ НЕСКОЛЬКО ПАПОК И ФАЙЛОВ
mk dir style
cd style
touch style.css

ВЕРНЕМСЯ НА ДИРЕКТОРИЮ ВЫШЕ
cd ..

ИНИЦИАЛИЗАЦИЯ В ЭТОЙ ПАПКЕ
git init

ДОБАВЛЕНИЕ В ИНДЕКС (КОГДА СОЗДАЕМ ФАЙЛЫ В ДИРЕКТОРИИ ИХ НУЖНО ДЛЯ ОТСЛЕЖИВАНИЯ ГИТ ДОБАВЛЯТЬ В ИНДЕКС STAGING AREA)
git add index.html / git add . (добавить один или все файлы в проекте)

ПОКАЗАТЬ СТАТУС ИЗМЕНЕННЫХ И ДОБАВЛЕННЫХ (НЕДОБАВЛЕННЫХ) В ИНДЕКС ФАЙЛОВ
git status (покажет что добавлено в индекс, зеленые добавлены в индекс)

УДАЛЕНИЕ ИЗ ИНДЕКСА
git rm --cached index.html
git status (теперь index.html будет показываться красным, то есть вне индекса)
git rm --cached -r . (удаление всех файлов из индекса)
(если без --cached то удаление файлов будет из индекса И рабочего каталога)

ДОБАВИМ ВСЕ СОДЕРЖИМОЕ ПАПКИ В ИНДЕКС git add .

ПОСЛЕ ДОБАВЛЕНИЯ ФАЙЛОВ В ИНДЕКС НЕОБХОДИМО СОЗДАТЬ КОММИТ ДЛЯ СОХРАНЕНИЯ ИЗМЕНЕНИЙ. КОММИТ - ЭТО ЗАФИКСИРОВАННОЕ СОСТОЯНИЕ ПРОЕКТА, В КОТОРОМ ВНЕСЕНЫ ВСЕ ИЗМЕНЕНИЯ + ИНФО ОБ АВТОРЕ, ВРЕМЕНИ СОЗДАНИЯ И ХЭШ. КОММИТЫ ОБРАЗУЮТ ИСТОРИЮ ИЗМЕНЕНИЙ ПРОЕКТА.
git commit -m “add project” (-m указывает краткое описание изменений в проекте)

ПОСМОТРЕТЬ ИСТОРИЮ ВСЕХ КОММИТОВ В РЕПОЗИТОРИИ
git log
git show (для просмотра более детальной информации)

РАБОТАЕМ С GITHUB
ДОБАВИТЬ SSH ключ для аутентификации на сервисе GitHub
ssh-keygen затем Enter три раза
Открываем ключ при помощи команды cat и копируем содержимое
Переходим на сайт github.com и нажимаем settings -> ssh and GPG keys -> new ssh keys -> в поле key вставляем наш ключ, указываем title и добавляем в профиль

Если удаленный репо присутствует а локальный нет: 
git clone <ссылка на репозиторий на github.com>
cd project (переходим в папку проекта)
ls -l (смотрим содержимое, проект добавлен)

Если есть локальный, но нет удаленного:
Создаем на github.com репозиторий
Переходим в папку локального проекта и пишем команду git remote add origin <ссылка на репозиторий на github.com>, где origin название глобального репозитория по умолчанию.
git add . (добавляем содержимое проекта в индекс)
git commit -m “add index” (создадим коммит)
git push -u origin master (origin master это <repo name>) (для отправки изменений на сервер github)

Если оба отсутствуют:
Переходим в папку cd proj
Создадим файл touch index.html
Добавим содержимое nano index.html
Пробуем отправить изменения на удаленный репо: 
git add .
git commit -m “add new commit”
git remote add origin <ссылка на репо.git> (создаем удаленный репо)
git push -u origin master (запушим проект) - проект отправлен

Отправка на доп репозиторий
Создаем репозиторий на github.com
git remote -v (появится список удаленных репозиториев с правами fetch и/или push)
git remote add <repo name> <URL> добавляем новый репо
git push -u <repo name> отправим изменения на новый репо

.config.json

nano file.log

nano .gitignore -> error.log или *.log (то есть все заканчив. на .log)
git status
КАК ИСКЛЮЧИТЬ ЦЕЛУЮ ПАПКУ
создадим папку logs mk dir logs 
перенесем файлы в нее mv *.log logs
git status
nano .gitignore -> logs/
git status
ОТРИЦАНИЕ ИГНОРИРОВАНИЯ nano .gitignore -> logs/*.log 
!log/main.log (теперь файл main.log исключен из игнора)
